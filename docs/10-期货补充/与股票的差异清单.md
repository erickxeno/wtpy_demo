## 差异总表

| 维度 | 股票（A 股） | 期货（国内商品/金融） | 工程/配置差异 |
|---|---|---|---|
| 交易制度 | T+1（当日买次日卖） | T+0（日内可反复） | `env.mocker: stk` vs `cta` / `hft`；CtaContext `stra_get_position(bonlyvalid=True)` 才区分可用 |
| 保证金 | 无，全额买入 | 有，10~15% 左右 | `ContractInfo.longMarginRatio/shortMarginRatio`；回测引擎自动算占用 |
| 杠杆 | 否 | 是（倒数的保证金比率） | 同上 |
| 做空 | 不可（融券除外） | 可 | 策略 `stra_enter_short` 会被股票 mocker 无视 |
| 手续费 | 佣金（双边）+ 印花税（卖单边 0.05%）+ 过户费 | 按手固定费 / 按金额比例；平今仓有特殊费率 | `fees_stk.json` vs `fees.json` 结构不同（期货有 `open/close_today/close_prev`） |
| 最低交易单位 | 100 股（1 手）；ETF 100 份 | 1 手（品种有不同 volscale） | `stk_comms.json` 里 `minlots=100`；期货 `commodities.json` 里各品种定义 |
| 涨跌停 | ±10%，ST ±5%，创/科 ±20% | 依品种（5~7%），夜盘另有规定 | `WTSTickStruct.upper_limit/lower_limit` 由 Parser 提供 |
| 交易时段 | 09:30–11:30 / 13:00–15:00 单日 | 日盘 + 夜盘，跨自然日 | `stk_sessions.json` 单段；期货 `sessions.json` 多段 + `offset!=0` |
| 集合竞价 | 9:15–9:25 (开) / 14:57–15:00 (收) | 部分品种有竞价 | `SessionInfo.auction` 字段 |
| 主力换月 | 无 | 有 | `hots.json` / `WtHotPicker` |
| 数据接入 | XTP / 华鑫 Torch / 米筐 / 券商内部 | CTP / Mini / Femas / Xele | Parser 模块名：`ParserXTP` vs `ParserCTP` |
| 柜台下单 | XTP / Torch | CTP | Trader 模块名：`TraderXTP` vs `TraderCTP` |
| Executer 算法 | 常 TWAP / VWAP 切大单 | MinImpact / 差额 | `executers.yaml.policy.default.name` |
| 交易日概念 | 自然日 = 交易日 | 夜盘 21:00 起算次日 | `offsetTime/originalTime` 就是为此 |
| 上市/退市 | 股票有 IPO / 退市 | 合约有挂牌/到期 | `ContractInfo.openDate/expireDate` |
| 分红派息 | 有，需复权 | 无 | `adjfactors.json` 只对股票 |
| 停牌 | 有 | 极少（异常） | 策略要判断 `stra_get_price` 为 0 或 Tick 空 |
| 代码体系 | `SSE.STK.600000` / `SSE.ETF.510300` | `SHFE.rb.HOT` / `SHFE.rb.2310` | `CodeHelper.stdCodeToStdCommID` |

## demo 层面的差异

| 文件 | 股票（cta_stk_bt） | 期货（cta_fut_bt） |
|---|---|---|
| `runBT.py` | `EngineType.ET_CTA` | 同 |
| `configbt.yaml.env.mocker` | `cta`（或 `stk`） | `cta`（或 `hft`） |
| `replayer.basefiles.commodity` | `stk_comms.json` | `commodities.json` |
| `replayer.basefiles.contract`  | `stocks.json` | `contracts.json` |
| `replayer.basefiles.hot`       | 无 | `hots.json` |
| `replayer.basefiles.session`   | `stk_sessions.json` | `sessions.json` |
| `replayer.fees`                | `fees_stk.json` | `fees.json` |
| 策略 `__is_stk__` | 常设 `True`（股票版 DualThrust） | `False` |

## 把股票策略"移植"到期货大致步骤

1. 把主合约代码改成期货 `SHFE.rb.HOT` 等。
2. 去掉 `isForStk=True`（或调相应分支）。
3. 把 yaml 的 4 个 basefiles + fees 切到期货版本。
4. 若涉及换月：`WtBtEngine.registerCustomRule` 或维护 `hots.json`。
5. 更新 Parser / Trader 配置（期货 CTP）。
6. 把 `stra_enter_short` 分支打开（期货允许做空）。
7. 回测验证，关注换月日的异常 PnL。

## 期货 Snooper / Analyst 注意点

- 回测报表里 **保证金占用** 是首等大事；Calmar 比股票更有参考意义。
- 夜盘跨交易日，需要 Analyst 理解跨日 fund 曲线（`funds.csv` 里多条记录可能在同一自然日）。
- 持仓字段 `WTSBarStruct.hold`（持仓量）股票里为 0，期货有意义。
