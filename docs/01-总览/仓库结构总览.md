## 一层全景

```
wtpy_demo/                     ← 本仓库根
├── bt/                        ← 我们的回测实战目录（输出/报表/数据）
├── docker/                    ← 我们写的 Docker 化启动脚本（含 WtBtSnooper）
├── docker-compose.yml
├── docs/                      ← 你在这里：中文阅读笔记（不触碰上游）
├── openspec/                  ← 本次阅读计划的规范文件
├── scripts/
└── wtpy/                      ← WonderTrader Python 子框架（上游 subtree 镜像）
    ├── README.md
    ├── demos/                 ← 各类运行样例
    ├── docs/                  ← 上游自带文档 (INDICATORS / JOURNAL)
    ├── setup.py / MANIFEST.in
    ├── tools/
    └── wtpy/                  ← 真正的 Python 包源码
```

> `wtpy/` 是 [subtree](../00-基础/术语表.md#subtreegit-子树)，上游更新时会整体拉取；所以我们**严禁在 `wtpy/` 内新增/修改任何文件**。

## `wtpy/wtpy/` ——核心 Python 包（重点阅读）

```
wtpy/wtpy/
├── __init__.py                ← 暴露 WtEngine / WtBtEngine / WtDtEngine / EngineType
├── apps/                      ← 应用层工具
│   ├── WtBtAnalyst.py         ← 回测绩效分析（必读）
│   ├── WtCtaOptimizer.py      ← 并行参数优化
│   ├── WtCtaGAOptimizer.py    ← 遗传算法优化
│   ├── WtHotPicker.py         ← 期货主力换月
│   ├── WtHftOptimizer.py
│   ├── WtCCLoader.py
│   └── datahelper/            ← tushare/rqdata/tqsdk/akshare 数据抓取适配
├── monitor/                   ← 监控/可视化服务
│   ├── WtMonSvr.py            ← Flask HTTP 服务端
│   ├── WtBtSnooper.py         ← 回测结果可视化（本仓库 docker 目录重点使用）
│   ├── WtBtMon.py             ← 回测集中管理
│   ├── DataMgr.py / PushSvr.py / WatchDog.py / EventReceiver.py
│   └── static/                ← web UI 静态文件
├── wrapper/                   ← 与 C++ 底层对接的 ctypes 包装
│   ├── WtBtWrapper.py         ← 回测引擎包装（单例）
│   ├── WtWrapper.py           ← 实盘引擎包装（单例）
│   ├── WtDtWrapper.py         ← 数据引擎包装（单例）
│   ├── WtDtServoApi.py        ← 查询 datakit 落地数据
│   ├── WtDtHelper.py          ← csv ↔ dsb/dmb 转换
│   ├── WtExecApi.py           ← 独立执行器 WtExecMon 对接
│   ├── WtMQWrapper.py         ← 消息队列包装
│   ├── ContractLoader.py      ← 从 CTP 拉取合约
│   ├── TraderDumper.py
│   ├── PlatformHelper.py
│   ├── x64/  x86/             ← Windows DLL
│   └── linux/                 ← Linux SO
├── StrategyDefs.py            ← 策略基类 BaseCtaStrategy / BaseHftStrategy / BaseSelStrategy（必读）
├── CtaContext.py              ← CTA 策略运行时 API（股票主轴必读）
├── HftContext.py              ← HFT 策略运行时 API
├── SelContext.py              ← SEL 选股策略 API
├── WtEngine.py                ← 实盘引擎门面
├── WtBtEngine.py              ← 回测引擎门面
├── WtDtEngine.py              ← 数据引擎门面
├── WtDtServo.py               ← 在用户脚本里查 datakit 落地数据
├── ContractMgr.py             ← 加载 stocks.json / contracts.json
├── ProductMgr.py              ← 品种（未上市合约模板）管理
├── SessionMgr.py              ← 交易时段模板管理
├── CodeHelper.py              ← 合约代码工具
├── ExtModuleDefs.py / ExtToolDefs.py  ← 扩展模块协议
├── WtCoreDefs.py              ← 核心数据结构（Python 镜像）
├── WtDataDefs.py / WtUtilDefs.py / WtMsgQue.py
```

**本次重点（★）**：`WtBtEngine.py`、`WtEngine.py`、`WtDtEngine.py`、`StrategyDefs.py`、`CtaContext.py`、`ContractMgr.py`、`SessionMgr.py`、`apps/WtBtAnalyst.py`、`monitor/WtBtSnooper.py`、`wrapper/WtBtWrapper.py`。

**暂可跳过**：`WtMsgQue.py`、`TraderDumper.py`、`ExtModuleDefs.py`（首次阅读不涉及）、`x86/`（现代平台默认 x64）。

## `wtpy/tools/`

仓库根的小脚本，常规开发不用；可跳过。

## `wtpy/docs/`

- `INDICATORS.md`：技术指标（被 `WtBtAnalyst` / `WtBtSnooper` 使用）文档。
- `JOURNAL.md`：上游历史变更日志。
- 这两份作为一手资料被 [参考链接](../99-附录/参考链接.md) 引用；我们**不搬运**、**不替代**。

## 本仓库根多出来的目录（非上游）

| 目录 | 用途 |
|---|---|
| `bt/` | 我们的实盘前期回测工作目录（产出 `outputs_bt/`、绩效 Excel、journal） |
| `docker/` | Dockerfile + `launch_snooper.py` + `gen_summary.py`；复用 `WtBtSnooper` 做可视化 |
| `scripts/` | 辅助脚本（抓数据、启动等） |
| `openspec/` | 本次阅读计划的规范与变更 |
| `docs/` | 你正在读的文档 |
