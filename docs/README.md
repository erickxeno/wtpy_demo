## 渐进式披露层级（L0 → L3）

| 层级 | 回答什么问题 | 典型篇幅 | 是否可贴源码 |
|---|---|---|---|
| **L0 概览** | 这是什么 / 为什么存在 | 1–2 页 | 否 |
| **L1 概念** | 由哪些核心概念构成 / 它们之间的关系 | 2–5 页，常配 mermaid 图 | 引用路径，不贴代码 |
| **L2 模块** | 某一模块（文件/类）做什么，公开接口与职责 | 3–8 页 | 引用 `path:L行号`，关键片段可贴 |
| **L3 源码级** | 对某 demo / 函数逐段讲解 | 5–15 页 | 允许大段贴代码并加注释 |

> 原则：**低层级不得假设读者读过高层级**；**高层级可以反链到低层级**作为前置。

## 每篇文档必带的 6 字段元信息头

除 `README.md`、`00-基础/术语表.md`、`99-附录/常见问题FAQ.md` 外，`docs/**/*.md` **必须**在正文前声明：

```markdown
---
层级: L0 | L1 | L2 | L3
前置阅读:
  - docs/xx/yy.md
本篇目标: 一句话
读完后应能回答:
  - 问题1
  - 问题2
关键源码:
  - wtpy/wtpy/xxx.py:L12-L45
术语表反链:
  - 术语A
  - 术语B
---
```

L0 文档 `关键源码` 可为空字符串或 `(无)`；L2/L3 至少要有 1 条 `wtpy/...path:Lxx-Lyy` 指向。

## 术语表使用规则

- **单一真理源**：只在 `00-基础/术语表.md` 内维护解释；其他文档通过反链引用，**不得**内联重复解释。
- **新增触发**：任何文档在产出过程中遇到新术语 → 先去术语表新增一条（中文解释 + `tag: 股票/期货/通用` + `首次出现` 反链），再在正文用 `[术语](../00-基础/术语表.md#锚点)` 引用。
- **核心 20 词**：CTA / HFT / SEL / Bar / Tick / K线 / DataKit / Parser / Executer / Trader / Broker / WtEngine / WtBtEngine / WtDtEngine / Context / Strategy / 回测 / 实盘 / T+1 / 滑点。
