## 二、"下一步该读哪篇" 决策树

```
你是谁？
├─ AI 冷启动 / 从未读过 wtpy
│   → 顺着 1. 骨架 → 2. 总览 → 3. 架构 → 4. 引擎 读完 L0/L1；
│     再按需挑 L2/L3。
│
├─ 只想跑通股票回测
│   → 4.2 (WtBtEngine) → 7.1 (configbt.yaml)
│     → 9.1 (cta_stk_bt 精读) → 7.2 (WtBtAnalyst 看结果)
│
├─ 要写一个 CTA 股票策略
│   → 5.1 (策略基类) → 5.2 (CtaContext) → 6.3 (stocks.json/T+1)
│     → 9.1 (参考 demo)
│
├─ 要理解数据从哪来、存哪
│   → 6.1 (dsb/dmb/csv) → 6.2 (WtDtServo) → 9.3 (datakit_stk demo)
│     → 附录 `docker/` 与 `bt/` 下的实战 journal
│
├─ 要上实盘
│   → 8.1 (实盘配置) → 9.2 (cta_stk 逐段) → 8.2 (monitor)
│
├─ 调参 / 优化
│   → 10.1 (CtaOptimizer)
│
├─ 对期货有兴趣
│   → 11.1 (与股票差异清单) → 从差异项反链回股票侧文档阅读
│
└─ 遇到生词
    → 直接打开 `00-基础/术语表.md`；查不到就顺手登记并补充。
```

## 三、进度更新规则

- 每完成一篇 md，即刻把对应行 `⬜` 改为 `✅`。
- 每新增一组术语，要在"对应任务行"后的"术语表补充"子项打勾。
- 跨会话恢复：打开本文件 → 找到第一条 `⬜` 的任务 → 跳到 `tasks.md` 该编号 → 开始。
- **不得**在本文件以外另建进度表。
