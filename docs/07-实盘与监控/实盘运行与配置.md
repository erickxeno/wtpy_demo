## 启动入口 `run.py`

```python
from wtpy import WtEngine, EngineType
engine = WtEngine(EngineType.ET_CTA)
engine.init('../common/', "config.yaml",
            commfile="stk_comms.json", contractfile="stocks.json")
engine.add_cta_strategy(StraDualThrust(
    name='pydt_SH600000', code="SSE.STK.600000",
    barCnt=50, period="d1", days=30,
    k1=0.1, k2=0.1, isForStk=True))
engine.run(True)
while True: time.sleep(1)
```

和回测 `runBT.py` 的 3 点差异：
1. `WtEngine`（不是 `WtBtEngine`）。
2. `config.yaml`（非 `configbt.yaml`）。
3. `run(True)` 异步启动；主线程自行 sleep 保活。

## `config.yaml` 主配置

见 `demos/cta_stk/config.yaml`：
```yaml
basefiles:   {commodity, contract, holiday, session}
data.store:  {module: WtDataStorage, path: ../storage/}
env:
  name: cta
  fees: ../common/fees_stk.json
  filters: filters.yaml
  product.session: TRADING
  riskmon:
    active: true
    module: WtRiskMonFact
    name: SimpleRiskMon
    base_amount: 5000000
    risk_scale: 0.3
    inner_day_active: true
    inner_day_fd: 20.0
    ...
executers: executers.yaml
parsers:   tdparsers.yaml
traders:   tdtraders.yaml
bspolicy:  actpolicy.yaml
notifier:  {active: true, url: ipc:///wtpy/wt_cat_stk.ipc}
```

### `env.riskmon`（组合级风控）

- `base_amount`：组合基础资金（虚拟锚定量；动态权益 = base_amount + 增量）。
- `risk_scale`：**真实下单 = 策略目标 × risk_scale**。用它在"测试阶段把仓位降到三成仓"安全上线。
- `inner_day_active` + `inner_day_fd`：日内从高点回撤 X% 触发"降仓"。
- `multi_day_*`：多日跟踪止损。

## `tdparsers.yaml`（行情解析器）

两种常见模式：
- `ParserShm`：和 `datakit_stk` **同机共享内存**读实时行情（`path: ../datakit_stk/exchange.membin`）。
- `ParserUDP`：跨机广播接收；`bport` 广播，`sport` 查询。
- `ParserXTP` / `ParserCTP` / `ParserXele` 等：直接接柜台行情，免 DataKit。

> 股票实盘用 ParserShm + DataKit 是推荐路径：DataKit 常驻采集，策略不直连行情源，隔离故障。

## `tdtraders.yaml`（交易通道）

示例（XTP 模拟）：
```yaml
traders:
- active: true
  id: simnow
  module: TraderXTP
  host: 120.27.164.69
  port: 6001
  client: 1
  user/pass/acckey: 账号凭据
  quick: true
  riskmon:    # Trader 层风控（熔断下单频率）
    policy:
      default:
        cancel_stat_timespan: 10
        cancel_times_boundary: 20
        cancel_total_limits: 470
        order_stat_timespan: 10
        order_times_boundary: 20
```

- `id` 要和 `executers.yaml` 的 `trader: simnow` 对齐。
- `module` 决定加载哪个 Trader dll：股票用 `TraderXTP`、期货 `TraderCTP` / `TraderMini`。
- `riskmon`：**Trader 层**风控偏操作监管（撤单次数/频率），防止被交易所封单。和 `env.riskmon`（组合级风控）**两个不同层面**。

## `executers.yaml`（信号执行器）

```yaml
executers:
- active: true
  id: exec
  trader: simnow
  scale: 1
  local: true
  policy:
    default:
      name: WtExeFact.WtMinImpactExeUnit
      offset: 0      # 跳数偏移
      expire: 5      # 订单超时秒数
      pricemode: 1   # -1 己方最优 / 0 最新价 / 1 对手价
      span: 500      # 下单时间间隔 ms
      byrate: false
      lots: 1
      rate: 0
  clear:
    active: false
    excludes: [...]
    includes: [...]
```

**每条 executer**：从 Engine 拿策略目标仓位 → 按 `policy` 算出具体委托 → 通过 `trader` 发到柜台。
- `scale`：把策略目标放大/缩小。
- `policy.default` 是回退规则；可对具体品种加 key（如 `SHFE.rb:` {...}）覆盖。
- `name: WtExeFact.WtMinImpactExeUnit`：内置"最小冲击执行单元"。也可换 TWAP/VWAP 等。

## `filters.yaml`（盘中干预）

常用字段：`code_filters`（某些合约黑白名单）、`executor_filters`（停用某 executer）、`strategy_filters`（冻结策略）。盘中修改保存后**热加载**，不用重启。

## `actpolicy.yaml`（动作策略）

配合 `bspolicy` 指向它，控制"买卖开平"组合规则（例：股票只有"买入开/卖出平"；期货可能有"双开双平"）。股票场景多用默认；期货自定义多。

## 启动最小清单

要把股票实盘跑起来至少：
1. **基础文件**：`../common/stocks.json`、`stk_comms.json`、`stk_sessions.json`、`holidays.json`、`fees_stk.json`。
2. **DataKit**：同机起 `datakit_stk/runDT.py` 常驻采集。
3. **改 `tdtraders.yaml`**：写入你的真实券商账号/key。
4. **改 `tdparsers.yaml`**：保留 `ParserShm` + DataKit membin 路径；禁用 UDP。
5. **改 `executers.yaml`**：`trader` 与上文 `id` 对齐；根据合约类型选择 `policy.default`。
6. **改 `config.yaml`**：视需要调整 `riskmon.risk_scale`（测试阶段可 0.1）、`notifier.url`。

## 常见坑

- **ParserShm 路径不对**：DataKit 的 `exchange.membin` 相对路径写错 → 策略永远收不到 Tick。
- **Trader 登录失败**：`tdtraders.yaml` 的 `host/port/key` 拼错；程序会在 `on_channel_lost` 里吼，但不会退出。
- **Exec scale=0**：`env.riskmon.risk_scale=0` 会把下单量都裁成 0，表面上策略在跑，实盘毫无委托。
- **未订阅合约**：DataKit 没订阅该股，策略 `on_bar` 不会触发。
- **双交易日**：期货夜盘跨日，股票无此问题；但凌晨启动服务要注意 `session.offset=0`。
